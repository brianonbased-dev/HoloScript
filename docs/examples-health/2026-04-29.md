## Error Categories Observed

### 1. Block-vs-property syntax drift (most common)
```
Line N:N: Expected COLON, got LBRACE
Line N:N: Expected identifier, got COLON
```
Files write `property_name { nested: value }` where the parser now expects `property_name: { nested: value }` with an explicit colon. Seen in: ar-foundation/*, agents/*, atoms/*, audio/*, lighting/*, volumetric/*.

### 2. Too-many-blocks / suspicious keyword
```
Line 0:0: Too many blocks in program
Line 0:0: Suspicious keyword detected: process
```
`.hs` files using multi-block layouts or shell-adjacent keywords. Seen in: agents/guard-patrol.hs, agents/iot-sensor-pipeline.hs, ai-agent.hs.

### 3. UI block numeric argument
```
Line N:N: Expected 'element' in ui block, got NUMBER
```
UI block syntax expects named elements; numeric positional args no longer valid. Seen in: affordances/01-interactive-controls.holo.

### 4. Inline `=` assignment in traits
```
Line N:N: Unexpected token: EQUALS
```
Old assignment syntax `property = value` inside trait blocks. Seen in: affordances/02-mechanical-systems.holo.

### 5. Identifier-followed-by-identifier (type syntax gone)
```
Line N:N: Expected string, got IDENTIFIER
Line N:N: Expected LBRACE, got IDENTIFIER
```
Files using `object TypedName "label"` two-identifier pattern. Seen in: autonomous-ecosystems/02*, 03*.

### 6. AT-sigil in unexpected position
```
Line N:N: Expected LBRACE, got AT
```
Trait decorator appearing inside a block that expects a sub-block opener. Seen in: audio/spatial-audio-showcase.holo.
