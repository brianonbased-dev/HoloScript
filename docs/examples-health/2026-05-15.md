## Pattern: `post_processing { bloom { } }` drift

Parser now requires property-assignment syntax inside `post_processing` blocks.
The `bloom { intensity: ... }` sub-block pattern (used in 30+ examples) is no longer valid.
This is widespread drift — filing as background awareness.

**Parser domain:** `domain-postfx`  
**Symptom:** `Expected COLON, got LBRACE` at every nested block opener  
**Affected examples:** quickstart/*, physics/advanced-physics-showcase, weather/01-particle-emitters, many more  
**Not filed as Phase 4 seed** (parser/compiler bug, not a trait composition gap — outside A-009 scope)

## Pattern: `hinge "Name" { }` articulation sub-block drift

Named sub-blocks inside `articulation { }` are no longer parsed. Valid syntax is now
`@joint { jointType: "hinge", ... }` applied to individual objects.

**Fixed in:** `examples/showcase/realistic-forest.refreshed.holo`
