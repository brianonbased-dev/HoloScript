## Findings

### P2-A â€” `/approve-via-tx` has no batch-size cap; unbounded outbound RPC calls

- **Commit**: `6f0cead`
- **File**: `packages/mcp-server/src/holomesh/routes/attestation-routes.ts`
- **Severity**: P2 (resource exhaustion via self-inflicted DoS)

The new `POST /api/identity/attestation/approve-via-tx` endpoint accepts either a single envelope or a batch via `attestations_via_tx: [...]`. For each envelope it makes **two parallel** `eth_getTransactionByHash` / `eth_getTransactionReceipt` calls to the Base mainnet RPC. There is no cap on `envelopes.length`.

A founder-authenticated request with 500 envelopes would fire 1,000 outbound RPC calls simultaneously, exhausting the RPC provider's rate window and potentially timing out the entire HoloMesh process. The `requireAuth + isFounder` guard prevents unauthorized use, but does not protect against accidental or adversarial over-batching from an authenticated founder session.

**Recommended fix** â€” add a guard before the loop:
```ts
const MAX_BATCH = 50;
if (envelopes.length > MAX_BATCH) {
  json(res, 400, { error: `batch too large (max ${MAX_BATCH}, got ${envelopes.length})` });
  return true;
}
```

### P2-B â€” `estimate_task_duration` MCP tool reads caller-supplied filesystem paths without boundary enforcement

- **Commit**: `435b53d`
- **File**: `packages/mcp-server/src/tools/estimate_task_duration.ts`
- **Severity**: P2 (OWASP A01 path traversal â€” scoped to MCP-authenticated sessions)

`estimateTaskDuration()` accepts `dataPath` and `sourceRoot` from the MCP tool call arguments. Both are passed to `firstExistingFile()` â†’ `resolve(p)` â†’ `existsSync` / `statSync` / `readFileSync` with **no boundary check**. A caller can supply `dataPath: "../../../../etc/some-world-readable.json"` and the tool will read and surface that file's contents if it parses as JSON matching `AgentTimeArtifact`.

Exploitability is limited to MCP-authenticated clients, but prompt-injection via LLM-driven tool calls is a realistic vector (the tool is surfaced to AI agents). The GOLD drive and previous review flagged a similar pattern (`fe186f8`, 2026-05-05 R-02) in `@holoscript/std/fs` â€” that fix should serve as the model here.

**Recommended fix** â€” add a `resolvedRoot.startsWith(allowedBase)` guard in `firstExistingFile`:
```ts
const ALLOWED_BASE = resolve(homedir(), '.ai-ecosystem');
function firstExistingFile(paths: Array<string | undefined>): string | null {
  for (const p of paths) {
    if (!p) continue;
    const resolved = resolve(p);
    if (!resolved.startsWith(ALLOWED_BASE + path.sep) && resolved !== ALLOWED_BASE) continue;
    try {
      if (existsSync(resolved) && statSync(resolved).isFile()) return resolved;
    } catch { /* skip */ }
  }
  return null;
}
```
