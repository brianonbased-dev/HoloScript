### P2 — Code Quality / Style Violations

#### P2-1: Shell injection risk in `paper-2-snn-bench.sh`
- **Commit**: `b2369fe` (feat snn-webgpu, mesh-deploy)
- **File**: `scripts/mesh-deploy/gate-bootstrappers/paper-2-snn-bench.sh`, line 136
- **Code**:
  ```bash
  ARTIFACT="$WORKSPACE/.bench-logs/paper-2-lif-throughput-automated.json"
  python3 -c "import json; d=json.load(open('$ARTIFACT')); ..."
  ```
- **Risk**: `$ARTIFACT` is derived from `PAPER2_WORKSPACE` (env var). If this variable contains a single-quote character, the shell interpolation breaks out of the Python string literal, enabling arbitrary Python code execution. In practice this script runs under controlled SSH sessions with trusted env vars, so the risk is low — but the pattern is wrong.
- **Fix**: Use `python3 -c "..." "$ARTIFACT"` with `sys.argv[1]` inside the script, or write a small helper `.py` file instead of an inline `-c` string.
- **Severity**: P2 (low exploitability in current usage, but a code quality violation that will bite when the bootstrapper is generalized).

#### P2-2: Extensive `any` casts in `HolographicSpriteTrait.ts`
- **Commit**: `ec40444` (feat holographic-sprite)
- **File**: `packages/core/src/traits/HolographicSpriteTrait.ts`
- **Count**: 14+ occurrences of `(node as any)` and `'@holographic_sprite' as any`
- **Violation**: CLAUDE.md style — prefer `unknown` over `any`. The `node as any` pattern is used extensively to call `.emit()` and `.on()` because `HSPlusNode` does not expose these methods in its type. The correct fix is to extend `HSPlusNode` with an event emitter interface (or use the typed emit/on patterns already established in other traits) rather than casting to `any` at each call site.
- **Impact**: Suppresses type errors that would catch mismatched event payloads or typos in event names.

#### P2-3: `any` casts and untyped dynamic import in `SegmentTrait.ts`
- **Commit**: `b700c42` (feat segment)
- **File**: `packages/core/src/traits/SegmentTrait.ts`
- **Issues**:
  - `let result: any` at line 217 — should be `unknown` with a discriminated union or type guard
  - `await import('@xenova/transformers' as any)` — using `as any` to bypass module resolution is unusual; the correct approach is `@ts-expect-error` with a comment explaining why, or adding a type declaration shim for the optional dependency
  - `(navigator as any).gpu.requestAdapter()` — should be typed via the WebGPU types package (`@webgpu/types`) rather than cast
  - 14+ `(node as any)` occurrences — same issue as P2-2 above

#### P2-4: Dynamic import of CDN URL cast to `any` in `SegmentTrait.ts`
- **Commit**: `b700c42`
- **File**: `packages/core/src/traits/SegmentTrait.ts`, line 160
- **Code**: `const mod = await import(cdnUrl as any);`
- **Risk**: Dynamic import of an arbitrary CDN URL cast to `any`. If the CDN URL can be influenced by untrusted input (e.g., config from a malicious `.holo` file), this is an XSS-equivalent in a Node/browser dual-target module. Currently `cdnUrl` is a hardcoded string constant in the module, so actual risk is P3 — but the pattern should use a typed URL string, not `any`.
