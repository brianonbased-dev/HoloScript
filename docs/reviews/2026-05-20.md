## Findings

### P1 — Security gate injection-pattern bypass too broad

**Commit**: `643d3cab` `fix(mcp-server): increase security gate limits for inline sourceFiles absorb`
**File**: `packages/mcp-server/src/security/gates.ts`

```typescript
// Current:
const resolvedGate1Config: Gate1Config =
  toolName === 'holo_absorb_repo' && Array.isArray(args['sourceFiles'])
    ? { ...(gate1Config ?? DEFAULT_GATE1_CONFIG), blockInjectionPatterns: false }
    : (gate1Config ?? DEFAULT_GATE1_CONFIG);
```

The check fires on `toolName === 'holo_absorb_repo' && Array.isArray(args['sourceFiles'])`. When true, `blockInjectionPatterns` is disabled for the **entire request**, including `rootDir`, `query`, and any other fields. The stated rationale (TypeScript source naturally contains `__proto__`, path-traversal strings, template literals) applies only to the string contents of `sourceFiles[*]`. The `rootDir` field is a filesystem path that should keep injection checking enabled.

An agent that passes even one `sourceFiles` element can now bypass injection checks on `rootDir` and all other args of that call.

**Recommended fix**: Apply the bypass only when building the string value under validation, not at the whole-request level. Alternatively, gate the injection check per-field: allow `sourceFiles[*]` to contain code patterns, but keep other fields checked.

This is not covered by a self-healing CI PR, so it is flagged here.

### P2 — `(window as any).AudioContext` in r3f-renderer scene components

**Commit**: `04f69f29` `feat(r3f-renderer): wire procedural audio bridges for 4 HoloShell scenes`
**Files**: `MountainLakeScene.tsx`, `NightCampfireScene.tsx`, `UnderwaterScene.tsx`, `WarmLibraryScene.tsx`

All four scenes use the same `(window as any).AudioContext || (window as any).webkitAudioContext` pattern. Proper typing:

```typescript
const AudioCtx =
  (window as Window & { AudioContext?: typeof AudioContext; webkitAudioContext?: typeof AudioContext })
    .AudioContext ??
  (window as Window & { webkitAudioContext?: typeof AudioContext })
    .webkitAudioContext;
```

Or extract a `getAudioContext()` helper once, shared across all scenes, rather than repeating the cast in four components.
