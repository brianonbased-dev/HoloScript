## Full Findings

### R-01 ┬╖ P1 ┬╖ `96e332e` тАФ Unsanitized template interpolation in `codeCreateObject`

**File:** `packages/studio/src/lib/brittney/BrittneyTools.ts`
**Location:** `codeCreateObject()` (~line 407)

```ts
if (opts.color) lines.push(`  color: "${opts.color}"`);
// ...
return code + `\n${type} "${name}" {\n${lines.join('\n')}\n}\n`;
```

`name`, `color`, and `primitive` are LLM-supplied strings inserted directly into generated `.holo` source without escaping quotes, backslashes, or newlines.

**Impact:** A value such as `name = 'foo"\nmalicious_block: true'` produces syntactically broken or structurally-injected HoloScript. The LLM is the source of `args.name`, so the risk is a confused/adversarial model call rather than end-user input тАФ but the `codeCreateObject` function operates at the text-generation boundary and should be hardened regardless.

**Fix:** Sanitize or quote-escape string fields before interpolation:
```ts
const escapeHoloString = (s: string) => s.replace(/\\/g, '\\\\').replace(/"/g, '\\"');
lines.push(`  color: "${escapeHoloString(opts.color)}"`);
return code + `\n${type} "${escapeHoloString(name)}" {\n${lines.join('\n')}\n}\n`;
```

### R-03 ┬╖ P2 ┬╖ `d2e74a3` тАФ `as any` + unguarded `NEXTAUTH_SECRET` in `api-auth.ts`

**File:** `packages/studio/src/lib/api-auth.ts`

Two issues in the new `getSession()` fallback:

1. **`as any` cast** (CLAUDE.md style violation тАФ use `unknown` or a proper type):
   ```ts
   req: { cookies: ... } as any,   // тЖР should be the actual next-auth/jwt Req shape
   ```
   The `getToken` function from `next-auth/jwt` accepts a specific `req` interface; the cast hides a type mismatch. Use `Parameters<typeof getToken>[0]['req']` instead.

2. **Unguarded `NEXTAUTH_SECRET`:**
   ```ts
   secret: process.env.NEXTAUTH_SECRET,  // could be undefined
   ```
   When `NEXTAUTH_SECRET` is not set, `getToken` receives `secret: undefined`. The library may silently skip signature verification in that case rather than throwing. Should guard:
   ```ts
   const secret = process.env.NEXTAUTH_SECRET;
   if (!secret) return null;  // or throw тАФ but fail fast, not silently
   ```

### R-04 ┬╖ P2 ┬╖ `d3e13ab` тАФ `require('fs')` in ESM HoloFileWatcher

**File:** `packages/studio/src/lib/hot-reload/HoloFileWatcher.ts` (~line 42)

```ts
import { watch, FSWatcher, existsSync } from 'fs';   // ESM import at top
// ...
this.isDirectory = existsSync(this.targetPath) && require('fs').statSync(this.targetPath).isDirectory();
//                                                   ^^^^^^^^^^ CJS require inside ESM file
```

`watch`, `existsSync`, and `FSWatcher` are already imported from `'fs'` at the module level. The `require('fs').statSync` call is redundant and CJS/ESM mixing that will throw `require is not defined` in a strict ESM context (e.g. `"type": "module"` in package.json or bundled with esbuild's ESM output).

**Fix:**
```ts
import { watch, FSWatcher, existsSync, statSync } from 'fs';
// ...
this.isDirectory = existsSync(this.targetPath) && statSync(this.targetPath).isDirectory();
```

### R-05 ┬╖ P2 ┬╖ `f4d49b2` тАФ API-sourced values interpolated into Brittney prompts

**File:** `packages/studio/src/components/ai/BrittneyBuildSurface.tsx` (~line 226)

```ts
const prompt = `Set up a HoloMesh "${t.name}" team session тАФ ${t.objective}.`;
```

`t.name` and `t.objective` come from a live `/api/holomesh/team/templates` fetch with no sanitization. If the HoloMesh backend were to serve crafted template data (misconfiguration, supply-chain, or SSRF), an attacker could manipulate Brittney's system prompt or inject instruction-following text that affects the AI's tool execution.

This is a prompt-injection surface, not a traditional XSS/CSRF risk, because the value reaches the LLM as conversation content.

**Fix:** Either (a) constrain the template data to a known set via a server-side allowlist, or (b) strip/limit the length of `name`/`objective` before interpolation, or (c) wrap the prompt in a structured message that makes it clear to the model the content is user-supplied data, not instructions.
