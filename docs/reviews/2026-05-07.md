## P0 — Security

### S-01 · `02be152` · Hardcoded wallet address in committed script

**File:** `scripts/anchor-founder-settlement-receipt.mjs:63`

```js
const FOUNDER_ANCHOR = '0x0C574397150Ad8d9f7FEF83fe86a2CBdf4A660E3';
```

A specific Ethereum wallet address is hardcoded as a constant in a committed script. While not a private key, address rotation requires a code change, and the address is embedded verbatim in HTML output written to `/.broadcaster/`. Should be sourced from an environment variable (`HOLOMESH_FOUNDER_ANCHOR`) or a non-committed config file.

**Action:** Replace with `process.env.HOLOMESH_FOUNDER_ANCHOR ?? (() => { throw new Error('HOLOMESH_FOUNDER_ANCHOR not set'); })()`.

---

### S-02 · `02be152` · Unbounded `fetch()` in RPC verifier

**File:** `scripts/anchor-founder-settlement-receipt.mjs:~417` (`rpc()` helper inside `verifyTx`)

The `rpc()` helper calls `fetch(RPC, { method: 'POST', ... })` with no `AbortController`/`signal`/timeout. The Base public RPC endpoint (`HOLOMESH_BASE_RPC`, defaulting to `https://mainnet.base.org`) can stall or reject connections without timing out. Two `rpc()` calls are made per verify — for `eth_getTransactionByHash` and `eth_getTransactionReceipt` — so a single hung connection blocks the process indefinitely.

**Action:** Wrap with a 30-second `AbortController` signal on each `fetch()` call.

## P1 — Broken Invariants

### I-01 · `98b94e0` · Orphaned chain tx when anchor broadcasts before state-machine write

**File:** `packages/mcp-server/src/holomesh/agent-negotiation.ts:~695`

`settleNegotiationWithAnchor` submits `anchorSettlement` to Base **before** calling `advanceNegotiation`. In a concurrent-settle race (another caller wins the same negotiation between the broadcast and the state-machine write), the on-chain tx is submitted and unrecoverable but the error result omits the tx hash, leaving no audit trail to correlate. The pre-broadcast state checks reduce probability but do not eliminate the race.

**Action:** Capture the tx hash in a `pendingAnchor` audit record before broadcasting; include it in all error paths from `advanceNegotiation`.

### I-02 · `5ce20a6` · `env_value_missing` conflates missing vs deliberately empty env var

**File:** `packages/mcp-server/src/holomesh/identity/vault-lease-registry.ts:~770`

When vault enforcement is ON and the lease is valid, `resolveSecretWithLease` throws `VaultLeaseError('env_value_missing')` for both `process.env[key] === undefined` and `process.env[key] === ''`. An operator who intentionally sets a key to `''` to signal "disabled" receives a configuration error indistinguishable from a missing lease.

**Action:** Distinguish the two cases with separate error codes (`env_not_set` vs `env_intentionally_empty`), or document in the VaultLeaseError that empty-string is treated as unset.

### I-03 · `07fb5f6` · Hex and scientific-notation strings bypass replica integer check

**File:** `packages/mcp-server/src/holomesh/routes/admin-routes.ts:~405`

The new validator uses `Number(replicasRaw)` to parse the input. `Number("0x10")` → 16, `Number("1e2")` → 100 — both pass `Number.isInteger()`. The original code used `parseInt(str, 10)` which rejected hex notation. The companion test (`0e949ab`) only covers `"3.5"`.

**Action:** Replace `Number(replicasRaw)` with `parseInt(replicasRaw, 10)` for string inputs; add test cases for `"0x10"`, `"1e2"`, `null`, and missing body.

### I-06 · `1272457` · `hash` parameter not format-validated in `getHologramAsset`

**File:** `packages/mcp-server/src/hologram-mcp-tools.ts:~640`

The `asset` parameter is correctly validated against an allowlist (path-traversal motivation visible in the commit). The `hash` parameter only receives a `typeof` / empty-string check before reaching `buildAssetUrl`. ### I-08 · `02be152` · Non-ECDSA placeholder strings in EIP-712 typed data

**File:** `scripts/anchor-founder-settlement-receipt.mjs:~114`

```js
initiatorSignature: '0xfounder-anchor-attestation-marathon-2026-05-06',
responderSignature: '0xfounder-anchor-attestation-marathon-2026-05-06',
```

These `0x`-prefixed descriptive strings are not valid ECDSA signatures. The `hashTypedData()` call computes an EIP-712 hash over this struct, producing a value that is incorrect relative to what a real signer would produce. Any downstream consumer performing signature recovery or format validation will silently fail or produce misleading output.

**Action:** Either gate the script with a `--dry-run` flag that skips `hashTypedData()`, or replace the placeholder with `ethers.ZeroHash` and document clearly that this is a non-signed attestation.
