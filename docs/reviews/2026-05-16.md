## P1 — Broken Invariant

### Q-01 · `9d1db03` · `as unknown as HSPlusNode` double-cast for new AST node types

**File:** `packages/core/src/parser/HoloScriptPlusParser.ts` (two new push sites)

Two new block types — `system` and `page` — are pushed into the composition AST via `as unknown as HSPlusNode`. Neither type is present in the `HSPlusNode` union, so the cast bypasses the type system entirely: any consumer traversing AST children and switching on `node.type` will hit an unhandled case. `explain_code`, `validate_holoscript`, and compiler pipelines that pattern-match on node types will silently skip or crash on these nodes.

**Recommendation:** Add `SystemNode` and `PageNode` (or a `ShellCompositionNode` union) to `HoloCompositionTypes.ts`, include them in the `HSPlusNode` discriminated union, and remove the double-casts. Tests for round-trip parse/validate of `system` and `page` blocks should be added alongside.

## P2 — Code Quality

### Q-02 · `5e779f4` · Missing content-length guard before regex on untrusted team messages

**File:** `packages/holoscript-agent/src/delegated-authority.ts`, `parseRequest()`

The regex `/\{[\s\S]*"protocol"\s*:\s*"delegated-authority\/v1"[\s\S]*\}/` is applied to `msg.content` without a length cap. Team messages are a semi-trusted boundary (any team member can send them). A pathologically long message — or one crafted to trigger catastrophic backtracking — will block the agent's async tick. There is also no guard preventing replay of old messages through the `processed` set growing unboundedly across long-running sessions.

**Recommendation:** Reject `msg.content` longer than a reasonable cap (e.g. 8 KB) before running the regex. Add a `maxProcessedIds` eviction policy (e.g. keep last 10 000 IDs in a bounded LRU) to prevent unbounded memory growth.

### Q-03 · `7edbc5d` · `NdjsonTrustStorage` accepts arbitrary `filePath` with no boundary check

**File:** `packages/core/src/trust/TrustLedger.ts`, `NdjsonTrustStorage` constructor

`path.resolve(filePath)` normalizes the path but does not restrict it to any sandbox directory. If `NdjsonTrustStorage` is ever instantiated with a value derived from user input (e.g. an MCP tool parameter), it becomes an arbitrary filesystem write primitive. The constructor is public and exported.

**Recommendation:** Either document in the type signature that `filePath` must be caller-controlled (internal only) and not exposed via MCP tools, or add a `baseDir` restriction that throws if the resolved path escapes it.

### Q-04 · `c2d1525` · HoloLand provisioning registries are in-memory with no persistence

**File:** `packages/mcp-server/src/hololand-mcp-tools.ts`, `playerRegistry` / `creatorRegistry` / `agentRegistry`

All three new provisioning registries are plain `Map` objects. A process restart (Railway redeploy, crash) silently loses every provisioned player, creator, and agent. Given the commit description ("HoloLand player/creator/agent provisioning path"), callers will treat provisioned entities as durable.

**Recommendation:** Either (a) persist to the same NDJSON/JSON store used by the existing `teamStore`/`agentKeyStore`, or (b) document clearly in the tool response that provisioning is session-scoped and callers must re-provision after restart.

### Q-05 · `6fc17a8` · Cast-over rather than fix: `stats.bySource as Record<string, number>`

**File:** `scripts/extract-grpo-prompts.ts`

Three `as Record<string, number>` and one `as string[]` cast were added to silence TypeScript errors after the `GRPOPromptExtractor` import path changed. The correct fix is to widen the return type of `GRPOPromptExtractor.getStats()` in `packages/absorb-service/src/self-improvement/GRPOPromptExtractor.ts`. The casts mask a type-definition gap that will silently fail if the shape of `stats` changes.

**Recommendation:** Align the return type in `GRPOPromptExtractor` to reflect the actual runtime shape, then remove the casts.
