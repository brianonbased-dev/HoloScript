- **File**: `packages/mcp-server/src/holomesh/routes/core-routes.ts:368`

The dead copy is not a correctness bug, but it will create maintenance confusion if the handler evolves — worth a follow-up cleanup task.

### P3-A — Marker-hiding via `String.fromCharCode` encodes meaning opaquely

### P3-B — `check-red-flags.mjs` `walk()` has no symlink cycle detection

- **Commit**: `9b7d576`
- **File**: `scripts/check-red-flags.mjs`
- **Severity**: P3 (edge-case reliability)

The `walk()` function recurses into subdirectories using `fs.readdirSync` with `withFileTypes: true`. It checks `entry.isDirectory()` but not `entry.isSymbolicLink()`. On a machine where a package symlinks back to a parent (common in pnpm workspace node_modules hoisting scenarios, or if someone creates a `packages/foo/src -> packages/bar/src` symlink), the walker could recurse infinitely. The script already excludes `node_modules` by name, which covers the most common case, but a direct symlink loop elsewhere would hang CI.

**Recommended fix**: add `if (entry.isSymbolicLink()) continue;` inside the walk loop, or use `fs.realpathSync` + a visited-set guard.

### P3-C — `detectCreatorAssetFormat()` uses a long sequential if-chain

- **Commit**: `20e29b8`
- **File**: `packages/studio/src/components/creator/creatorAssets.ts:44-57`
- **Severity**: P3 (style — minor maintainability)

The function has 9 sequential `if (ext === '...')` checks that mirror the `CreatorAssetFormat` union exactly. A lookup table avoids the need to keep the if-chain in sync with the type:

```ts
const EXT_TO_FORMAT: Record<string, CreatorAssetFormat> = {
  glb: 'glb', gltf: 'gltf', fbx: 'fbx', mp4: 'mp4',
  png: 'png', jpg: 'jpg', jpeg: 'jpeg', wav: 'wav', usdz: 'usdz',
};

export function detectCreatorAssetFormat(name: string): CreatorAssetFormat {
  const ext = name.split('.').pop()?.toLowerCase();
  return (ext && EXT_TO_FORMAT[ext]) || 'unknown';
}
```
