# USD Physics Export — NVIDIA Omniverse & Isaac Sim

HoloScript outputs physics-bearing USD ASCII (`.usda`) via `USDPhysicsCompiler`, targeting NVIDIA Omniverse and Isaac Sim natively. This is the primary path for factory digital twins, robot sim-to-real workflows, and AI training environments.

---

## Quick Start

### MCP Tool

```bash
# Isaac Sim (default context)
compile_to_usd composition=MyRobot targetContext=isaac_sim

# NVIDIA Omniverse
compile_to_usd composition=MyFactory targetContext=omniverse
```

### TypeScript API

```typescript
import { USDPhysicsCompiler } from '@holoscript/core/compiler';

const compiler = new USDPhysicsCompiler({ targetContext: 'isaac_sim' });
const usda = compiler.compile(composition, agentToken, 'robot.usda');
```

### CLI

```bash
holoscript compile --target usd --usd-context isaac_sim robot.holo -o robot.usda
holoscript compile --target usd --usd-context omniverse factory.holo -o factory.usda
```

---

## Target Contexts

| Context | Use case | Differences |
|---------|----------|-------------|
| `isaac_sim` (default) | Sim-to-real, RL training, Isaac Lab | Enforces translate→rotateXYZ→scale xformOp order; emits PhysX `ArticulationView` solver hints; Z-up by default |
| `omniverse` | Factory twins, visualization, USD Composer | Standard USD + PhysX schemas; fewer Isaac Lab constraints |
| `generic` | Any USD-compatible runtime | Portable PhysX schemas, no runtime-specific hints |

## Output Example

A rigid-body sphere + ground plane in HoloScript:

```holo
world PhysicsTest {
  object Ball {
    geometry: sphere
    scale: [0.2, 0.2, 0.2]
    position: [0, 0, 2]
    @physics { mass: 1.0, restitution: 0.7 }
  }
  object Ground {
    geometry: box
    scale: [10, 10, 0.1]
    position: [0, 0, 0]
    @physics { mass: 0, static: true }
  }
}
```

Compiled output (`compile_to_usd targetContext=isaac_sim`):

## Physics Schemas

HoloScript traits map directly to USD Physics schemas:

| HoloScript trait / annotation | USD schema emitted |
|-------------------------------|-------------------|
| `@physics { mass, restitution, friction }` | `PhysicsMassAPI`, `PhysicsMaterialAPI` |
| `@physics { rigid: true }` | `PhysicsRigidBodyAPI` |
| `@physics { collision: true }` (default) | `PhysicsCollisionAPI` |
| `@robot` / `@articulation` | `PhysicsArticulationRootAPI` |
| `@joint { type: hinge\|revolute }` | `PhysicsRevoluteJoint` |
| `@joint { type: prismatic\|slider }` | `PhysicsPrismaticJoint` |
| `@joint { type: ball\|spherical }` | `PhysicsSphericalJoint` |
| `@joint { type: fixed }` | `PhysicsFixedJoint` |
| `@joint { type: d6\|6dof }` | `PhysicsD6Joint` |
| `@drive { stiffness, damping }` | `PhysicsDriveAPI` |

### PhysicsScene defaults (Isaac Sim context)

```usda
physxScene:enableGPUDynamics = true
physxScene:timestep = 0.01667       # 60 Hz
physxScene:solverType = 1           # TGS solver
```

Override via compiler options:

```typescript
new USDPhysicsCompiler({
  targetContext: 'isaac_sim',
  gravity: [0, 0, -9.81],
  physicsTimestep: 1/120,           // 120 Hz
  enableGPUDynamics: false,         // CPU fallback
});
```

## Robot / Articulation Workflow

For robot compositions with joints and drives, set `enableArticulation: true` (default). The compiler wraps all links under a single `PhysicsArticulationRootAPI` prim:

```holo
robot ArmRobot {
  link Base { geometry: cylinder, scale: [0.3, 0.3, 0.1] }
  link Shoulder { geometry: cylinder, scale: [0.1, 0.1, 0.4], position: [0, 0, 0.3] }
  joint ShoulderJoint {
    parent: Base, child: Shoulder
    type: revolute, axis: Y
    @drive { stiffness: 1000, damping: 100, targetPosition: 0 }
  }
}
```

Compiled as:

```usda
def Xform "RobotArticulation" (
    prepend apiSchemas = ["PhysicsArticulationRootAPI"]
)
{
    bool physxArticulation:enabledSelfCollisions = false
    uint physxArticulation:solverPositionIterationCount = 8
    uint physxArticulation:solverVelocityIterationCount = 4

    def Xform "Base" (
        prepend apiSchemas = ["PhysicsRigidBodyAPI", "PhysicsCollisionAPI"]
    ) { ... }

    def Xform "Shoulder" (
        prepend apiSchemas = ["PhysicsRigidBodyAPI", "PhysicsCollisionAPI"]
    ) { ... }

    def RevoluteJoint "ShoulderJoint" (
        prepend apiSchemas = ["PhysicsDriveAPI:angular"]
    )
    {
        rel physics:body0 = </ArmRobot/RobotArticulation/Base>
        rel physics:body1 = </ArmRobot/RobotArticulation/Shoulder>
        uniform token physics:axis = "Y"
        float drive:angular:physics:stiffness = 1000
        float drive:angular:physics:damping = 100
        float drive:angular:physics:targetPosition = 0
    }
}
```

> **Isaac Lab / ArticulationView note**: The compiler enforces `xformOp` ordering (translate → rotateXYZ → scale) required by `ArticulationView.get_joint_positions()`. Using `targetContext: 'omniverse'` or `'generic'` relaxes this constraint.

## Semantic AST Embedding (Metaverse Standards Forum Interop)

By default (`embedSemanticAST: true`), the compiler injects the HoloScript semantic AST as `customData` on the root Xform. This enables round-trip behavioral metadata and is aligned with the Metaverse Standards Forum interoperability model:

```usda
def Xform "MyScene" (
    kind = "component"
    customData = {
        dictionary holoscript:semanticAST = {
            string sourceVersion = "6.0"
            string compositionName = "MyScene"
            ...
        }
    }
)
```

Disable if the downstream USD consumer doesn't expect custom metadata:

```typescript
new USDPhysicsCompiler({ embedSemanticAST: false });
```

## Provenance Hash (Paper 10)

For simulation evidence workflows, attach a provenance hash to the USDA output:

```typescript
const hash = computeSimulationHash(composition);  // SHA-256 of inputs
const compiler = new USDPhysicsCompiler({
  targetContext: 'isaac_sim',
  provenanceHash: hash,
});
```

Output includes a `# Provenance Hash: <sha256>` comment immediately after the file preamble. This enables `SimulationContract` receipt linkage — the hash identifies the exact composition state that produced the USD file.

## Import into NVIDIA Isaac Sim

### USD Physics file

1. Open Isaac Sim (2023.1.1+)
2. **File → Open** → select `scene.usda`
3. In the **Stage** panel, verify the `PhysicsScene` prim is present
4. **Physics → Play** to run simulation

### Via Python / Isaac Lab

```python
from omni.isaac.core import World
import omni.usd

# Load HoloScript-generated USD
world = World()
world.scene.add_default_ground_plane()

stage = omni.usd.get_context().get_stage()
stage.GetRootLayer().InsertSubLayerPath("robot.usda")
```

### Articulation in Isaac Lab

```python
from omni.isaac.lab.assets import ArticulationCfg
import omni.isaac.lab.sim as sim_utils

robot_cfg = ArticulationCfg(
    prim_path="/World/ArmRobot",
    spawn=sim_utils.UsdFileCfg(usd_path="arm_robot.usda"),
    init_state=ArticulationCfg.InitialStateCfg(
        joint_pos={"ShoulderJoint": 0.0},
    ),
)
```

## Import into NVIDIA Omniverse (USD Composer)

1. Open **NVIDIA Omniverse Launcher → USD Composer**
2. **File → Open** → select `factory.usda`
3. Use **Physics → Scene Settings** to verify `PhysicsScene` gravity and timestep
4. **Simulation → Play** to run

For Omniverse extensions:

```python
import omni.kit.commands

omni.kit.commands.execute(
    "CreateReferenceCommand",
    usd_context=omni.usd.get_context(),
    path_to="/World/HoloScriptScene",
    asset_path="factory.usda",
)
```

## Compiler Options Reference

```typescript
interface USDPhysicsCompilerOptions {
  targetContext?: 'isaac_sim' | 'omniverse' | 'generic';  // default: 'isaac_sim'
  upAxis?: 'Y' | 'Z';                   // default: 'Z'
  metersPerUnit?: number;               // default: 1.0
  timeCodesPerSecond?: number;          // default: 60
  gravity?: [number, number, number];   // default: [0, 0, -9.81]
  physicsTimestep?: number;             // default: 1/60
  enableGPUDynamics?: boolean;          // default: true
  includePhysicsScene?: boolean;        // default: true
  includeCollision?: boolean;           // default: true
  includeVisual?: boolean;              // default: true
  enableArticulation?: boolean;         // default: true
  defaultMass?: number;                 // default: 1.0
  defaultStaticFriction?: number;       // default: 0.5
  defaultDynamicFriction?: number;      // default: 0.5
  defaultRestitution?: number;          // default: 0.1
  embedSemanticAST?: boolean;           // default: true
  provenanceHash?: string;              // optional SHA-256 for SimulationContract
  stageName?: string;                   // default: 'HoloScriptPhysicsStage'
}
```

## Relationship to URDF/SDF Isaac Sim Path

HoloScript has two Isaac Sim export paths:

| Path | Format | Best for |
|------|--------|----------|
| `compile_to_usd targetContext=isaac_sim` | USD ASCII (.usda) | Native USD scenes, factory twins, multi-body environments, Omniverse interop |
| `compile_to_urdf --isaac-sim` | URDF XML | Single robot import via Isaac Sim URDF Importer, ROS 2 integration |

For robot development targeting Isaac Lab's `ArticulationView`, both paths work. USD is preferred for new workflows; URDF is better for ROS 2 ecosystem compatibility. See [`ISAAC_SIM_OPTIMIZATION.md`](./ISAAC_SIM_OPTIMIZATION.md) for the URDF path.
