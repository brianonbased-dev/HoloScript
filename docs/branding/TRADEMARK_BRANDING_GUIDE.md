## Usage Guidelines

### DO's ✅

1. **Always capitalize trademarks correctly**
   - ✅ "HoloScript" not "holoscript" or "Holoscript"
   - ✅ "StoryWeaver Protocol" not "storyweaver protocol" or "Story Weaver"
   - ✅ "VRR" not "vrr" or "Vrr"

2. **Use proper attribution for third-party trademarks**
   - ✅ "Coinbase AgentKit SDK" on first mention
   - ✅ "Zora Protocol" with proper capitalization
   - ✅ Include trademark notices in documentation

3. **Use trademarks as adjectives, not nouns**
   - ✅ "HoloScript language" not "a HoloScript"
   - ✅ "StoryWeaver Protocol system" not "a StoryWeaver"
   - ✅ "AgentKit SDK" not "an AgentKit"

4. **Include ™ or ® symbols in official documentation**
   - ✅ "HoloScript™" in marketing materials
   - ✅ Check if third-party marks require ® symbol

### DON'Ts ❌

1. **Never modify trademarks**
   - ❌ Don't pluralize: "HoloScripts"
   - ❌ Don't make possessive: "HoloScript's features" → use "HoloScript language features"
   - ❌ Don't abbreviate: "HS" for HoloScript

2. **Never use trademarks generically**
   - ❌ "We need to holoscript this" → use "write HoloScript code"
   - ❌ "StoryWeaver it" → use "generate with StoryWeaver Protocol"

3. **Never combine internal and third-party marks**
   - ❌ "HoloScript-Zora Integration" → use "HoloScript integration with Zora Protocol"
   - ❌ "AgentKit-HoloHub Connector" → use "HoloHub connector for AgentKit SDK"

## Capitalization & Formatting Rules

### Product Names (Title Case)

```
✅ HoloScript
✅ HoloScript Plus
✅ HoloHub
✅ StoryWeaver Protocol
✅ Virtual Reality Reality (VRR)
✅ Hololand
```

### File Extensions (Lowercase)

```
✅ .hs
✅ .hsplus
✅ .holo
```

### Package Names (Kebab-case with scope)

```
✅ @holoscript/core
✅ @holoscript/runtime
✅ @holoscript/vscode-extension
✅ @holoscript/llm-provider
```

### Acronyms (All Caps)

```
✅ VRR (Virtual Reality Reality)
✅ AR (Augmented Reality)
✅ VR (Virtual Reality)
✅ NFT (Non-Fungible Token)
✅ SDK (Software Development Kit)
✅ API (Application Programming Interface)
```

### Code Identifiers (CamelCase/snake_case)

```typescript
// Classes: PascalCase
class HoloScriptRuntime {}
class StoryWeaverService {}

// Functions/Methods: camelCase
function generateNarrative() {}
function createVRRTwin() {}

// Constants: SCREAMING_SNAKE_CASE
const HOLOSCRIPT_VERSION = '2.1.0';
const STORYWEAVER_API_ENDPOINT = 'https://...';

// Traits: snake_case with @
@weather_sync
@x402_paywall
@storyweaver_narrative
```

## Quick Reference

### ✅ Correct Usage

```markdown
**Example Documentation:**

HoloScript™ is a language for building VR/XR experiences. The HoloScript Plus
variant includes advanced VR traits. Creators can monetize their work through
HoloHub™, minting NFTs via Zora Protocol integration.

The Hololand™ platform uses a 3-layer architecture: AR entry points, VRR™
digital twins, and premium VR worlds. StoryWeaver Protocol™ generates AI
narratives using OpenAI GPT-4.

Payment flows use the x402 Protocol with Coinbase AgentKit SDK for AI agent
wallets on Base L2.
```

### ❌ Incorrect Usage

```markdown
**Example (DO NOT USE):**

holoscript is a vr language. Holoscript plus has vr traits. Use holohub to sell
NFTs on zora.

Hololand has AR->vrr->VR layers. Storyweaver uses gpt-4 AI.

X402 payments use agentkit wallets on BASE.
```
