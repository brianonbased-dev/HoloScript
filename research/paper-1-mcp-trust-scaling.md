## Summary

Trace size scales **linearly** with the number of simulation entries (R² = 1.000000).  
The measured slope is **≈ 282 bytes/entry** under the current CAEL v1 serialiser.

> **Discrepancy with published abstract**: The paper originally stated "~180 bytes per
> entry." That figure was derived from the pre-Wave-2 schema (no per-step `stateDigests`
> payload). Wave-2 item 5a (2026-04-20) added FNV-1a state-vector digests to every step
> entry, adding ~100 bytes/entry. The paper's §Trace Size table and abstract must be
> updated to reflect the current implementation.

## Root Cause: Wave-2 State Digests

The original 180 bytes/entry estimate assumed this step payload:

```json
{ "wallDelta": 0.01, "stepsTaken": 1, "totalSteps": 1 }
```

The current implementation (Wave-2 item 5a) adds:

```json
{ "wallDelta": 0.01, "stepsTaken": 1, "totalSteps": 1, "stateDigests": ["a3f4e1b2"] }
```

Each FNV-1a digest is 8 hex chars. With JSON key and array delimiters:
`"stateDigests":["a3f4e1b2"]` ≈ 24 bytes. Additional payload growth from longer
numeric timestamp and simTime values accounts for the remaining ~58 bytes difference.
