## Critical Findings

### 1. Shell Env Shadowing `.env` File (P0)

**Problem**: Shell environment variables shadow the real API key from `~/.ai-ecosystem/.env`:

```bash
# Shell env (BAD — overrides .env file)
ANTHROPIC_API_KEY=                        # EMPTY
ANTHROPIC_BASE_URL=http://127.0.0.1:11434 # Redirects to Ollama
ANTHROPIC_AUTH_TOKEN=ollama
ANTHROPIC_DEFAULT_OPUS_MODEL=kimi-k2.6:cloud
ANTHROPIC_DEFAULT_SONNET_MODEL=kimi-k2.6:cloud
ANTHROPIC_DEFAULT_HAIKU_MODEL=kimi-k2.6:cloud
```

**Impact**: Every Anthropic SDK call in studio (Brittney, absorb, etc.) fails with "model not found" because requests go to Ollama instead of api.anthropic.com.

**Root cause**: These vars are injected by shell startup (likely `.bashrc` or Windows env). They are NOT in `~/.ai-ecosystem/.env`.

**Fix**: Unset or correct these shell variables. The `.env` file has the real key.

```bash
unset ANTHROPIC_BASE_URL
unset ANTHROPIC_AUTH_TOKEN
unset ANTHROPIC_DEFAULT_OPUS_MODEL
unset ANTHROPIC_DEFAULT_SONNET_MODEL
unset ANTHROPIC_DEFAULT_HAIKU_MODEL
# Then source the real env
source ~/.ai-ecosystem/.env
```

### 2. Missing `NEXTAUTH_SECRET` in Studio Dev (P0)

**Problem**: NextAuth v4 `getServerSession()` fails silently when `NEXTAUTH_SECRET` is missing. Studio auth returns empty sessions.

**Evidence**: `api-auth.ts:30` uses `process.env.NEXTAUTH_SECRET` for `getToken()` fallback.

**Fix**: Export `NEXTAUTH_SECRET` before starting dev server:
```bash
export NEXTAUTH_SECRET=$(grep NEXTAUTH_SECRET ~/.ai-ecosystem/.env | cut -d= -f2)
```

### 3. Wrong `NEXTAUTH_URL` in Example (P1)

**Problem**: `packages/studio/.env.local.example` has `NEXTAUTH_URL=http://localhost:3000` but:
- Studio dev server runs on port 3101 (`pnpm run dev --port 3101`)
- Studio production is `https://holoscript.studio`

**Fix**: Update example to match actual dev port.

### 4. Missing `.env.local` in Studio (P1)

**Problem**: No actual `.env.local` exists in `packages/studio/`. Dev server relies entirely on shell env vars which are polluted.

**Fix**: Create `packages/studio/.env.local` with correct values.

### 5. Inconsistent Model Names (P1)

**Problem**: `packages/studio/.env.local.example` has `ANTHROPIC_MODEL=claude-sonnet-4-5-20250929` but:
- Brittney route default is `claude-opus-4-7`
- Real available models: `claude-opus-4-7`, `claude-sonnet-4-6`, `claude-haiku-4-5-20251001`
- The example model name is not in the Anthropic model list

**Fix**: Update example to use valid model names.
