# v5.3 "Tooling as Semantic Bridge" — Showcase

**Time to First HoloScript World (TTFHW): < 5 minutes**

This walkthrough demonstrates the two new MCP tools added in v5.3:
`suggest_universal_traits` and `generate_service_contract`.

---

## Step 1: Suggest Universal Traits

Use the `suggest_universal_traits` tool to discover which v6 traits apply
to your service architecture.

```json
{
  "tool": "suggest_universal_traits",
  "arguments": {
    "description": "REST API with JWT auth, Redis caching, and Kafka event pipeline",
    "context": "microservice for user management"
  }
}
```

**Response** (example):
```json
{
  "traits": [
    "@rest_resource", "@endpoint", "@service",
    "@jwt_config", "@jwt_verifier",
    "@cache", "@key_value_store",
    "@message_broker", "@topic", "@subscription",
    "@pipeline", "@stream",
    "@health_check",
    "@retry", "@circuit_breaker",
    "@structured_log"
  ],
  "domains": {
    "service": ["@rest_resource", "@endpoint", "@service"],
    "network": ["@jwt_config", "@jwt_verifier"],
    "data": ["@cache", "@key_value_store"],
    "pipeline": ["@message_broker", "@topic", "@subscription", "@pipeline", "@stream"],
    "metric": ["@health_check", "@structured_log"],
    "resilience": ["@retry", "@circuit_breaker"]
  },
  "confidence": 0.95
}
```

The tool covers **8 domains**: service, contract, data, network, pipeline,
metric, container, resilience. Cross-domain inference auto-adds
`@health_check` (for services), `@retry`/`@circuit_breaker` (for pipelines),
and `@structured_log` (for data operations).

## Step 2: Generate Service Contract

Feed an OpenAPI spec (JSON or YAML) to `generate_service_contract` and get
a `.holo` composition with full trait annotations.

```json
{
  "tool": "generate_service_contract",
  "arguments": {
    "input": "{\"openapi\":\"3.0.3\",\"info\":{\"title\":\"User API\",\"version\":\"1.0.0\"},\"paths\":{\"/users\":{\"get\":{\"operationId\":\"listUsers\",\"summary\":\"List users\"},\"post\":{\"operationId\":\"createUser\",\"summary\":\"Create user\",\"requestBody\":{\"content\":{\"application/json\":{\"schema\":{\"$ref\":\"#/components/schemas/User\"}}}}}},\"/users/{id}\":{\"get\":{\"operationId\":\"getUser\",\"summary\":\"Get user\"}}},\"components\":{\"schemas\":{\"User\":{\"type\":\"object\",\"required\":[\"id\",\"name\"],\"properties\":{\"id\":{\"type\":\"string\"},\"name\":{\"type\":\"string\"},\"email\":{\"type\":\"string\"}}}}}}"
  }
}
```

The tool auto-detects the format and generates valid `.holo` output:
- `@schema` + `@contract` blocks for each schema
- `@endpoint` + `@handler` blocks for each path/method
- `@service` + `@rest_resource` metadata block
- Field types mapped from OpenAPI → HoloScript types

TypeScript interfaces work too:
```json
{
  "tool": "generate_service_contract",
  "arguments": {
    "input": "interface User { id: string; name: string; email?: string; }",
    "format": "typescript"
  }
}
```

---

## Step 3: Explain the Contract

Use `explain_service_contract` to get a human-readable summary of any
`.holo` service composition:

```json
{
  "tool": "explain_service_contract",
  "arguments": {
    "code": "... (paste .holo code from Step 2)"
  }
}
```

Returns endpoint details, schema names, and trait counts.
