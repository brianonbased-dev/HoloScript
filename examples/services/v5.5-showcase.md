# v5.5 "Agents as Universal Orchestrators" Showcase

## What's New

v5.5 connects HoloScript's agent infrastructure into a coherent orchestration layer. Agents can discover each other across compositions, delegate tasks, and compose multi-step workflows — all driven by LLMs through 5 new MCP tools.

### New Infrastructure
| Component | Purpose |
|-----------|---------|
| FederatedRegistryAdapter | Cross-composition agent discovery via `/.well-known/agent-card.json` |
| TaskDelegationService | Cross-agent task forwarding (local + remote A2A JSON-RPC) |
| SkillWorkflowEngine | DAG-based skill composition with parallel execution |
| OrchestratorAgent | First concrete `BaseAgent` (7-phase uAA2++ protocol) |

### New MCP Tools
| Tool | Description |
|------|-------------|
| `discover_agents` | Find agents by capability, domain, or tags |
| `delegate_task` | Send a task to a discovered agent (auto-select or by ID) |
| `get_task_status` | Check the status of a delegated task |
| `compose_workflow` | Define and validate a multi-step skill workflow DAG |
| `execute_workflow` | Execute a validated workflow with step-by-step results |

## Walkthrough: 5 Minutes to Multi-Agent Pipeline

### Step 1: Discover Agents

Find all agents with IoT analysis capabilities:

```
discover_agents({
  type: "analyze",
  domain: "iot"
})
```

Returns: `{ agents: [{ id: "sensor-agent-01", score: 0.85, ... }], total: 1 }`

### Step 2: Delegate a Task

Send a task directly to a known agent:

```
delegate_task({
  agentId: "analytics-agent-01",
  skillId: "aggregate_readings",
  arguments: { readings: [...], window_ms: 60000 }
})
```

Or auto-delegate by capability:

```
delegate_task({
  type: "transform",
  domain: "iot",
  skillId: "aggregate_readings",
  arguments: { readings: [...], window_ms: 60000 }
})
```

### Step 3: Compose a Workflow

Define a multi-step pipeline as a DAG:

```
compose_workflow({
  name: "sensor-to-dashboard",
  steps: [
    { id: "collect", skillId: "read_sensors",
      inputs: { device_ids: ["temp-01"] } },
    { id: "analyze", skillId: "aggregate_readings",
      dependsOn: ["collect"],
      inputs: { readings: { type: "ref", stepId: "collect", outputKey: "readings" } } },
    { id: "render", skillId: "render_chart",
      dependsOn: ["analyze"],
      inputs: { data: { type: "ref", stepId: "analyze", outputKey: "summary" } } }
  ]
})
```

Returns: `{ valid: true, stepCount: 3, executionPlan: [["collect"], ["analyze"], ["render"]] }`

### Step 4: Execute the Workflow

```
execute_workflow({
  name: "sensor-to-dashboard",
  steps: [ ...same steps as above... ]
})
```

Each step runs through the local MCP tool executor. Results flow between steps via `ref` inputs.

### Step 5: Run an Orchestrator Cycle

The OrchestratorAgent runs a full 7-phase uAA2++ cycle:

```typescript
const orchestrator = new OrchestratorAgent({
  name: "IoT Controller",
  domain: "iot",
  autoDiscovery: false,
});

const result = await orchestrator.runCycle("aggregate_readings", {
  type: "transform",
  domain: "iot",
});

// result.phases = [INTAKE, REFLECT, EXECUTE, COMPRESS, REINTAKE, GROW, EVOLVE]
// result.status = "complete"
```

## Architecture

```
/.well-known/agent-card.json    Remote Agent Cards
        |                              |
        v                              v
  FederatedRegistryAdapter  ------>  AgentRegistry
        |                              |
        v                              v
  TaskDelegationService     <---  CapabilityMatcher
        |                              |
        v                              v
  SkillWorkflowEngine       <---  OrchestratorAgent
        |                         (7-phase uAA2++)
        v
  MCP Tool Executor
  (discover / delegate / compose / execute)
```
